1) ->Git is a free, distributed high version control system suitable for tracking modifications in source code in software development.
   ->GitHub is a web-based Git repository which has a cloud based storage. It offers all distributed version control and source code
     management functionality of Git which makes it easier to collaborate with Git.

2) i) Git config:- This command sets the author name and email address respectively to be used with your commits.
  ii) Git init:- This command is used to start a new repository.
 iii) Git clone:- This command is used to obtain a repository from the existing URL.
  iv) Git add:- This command adds a file to the staging area.
   v) Git commit:- This command records or snapshots the file permanently in the version history.
  vi) Git branch:- This command lists all the local branches in the current repository.

3) i) Switch to the branch that you want to create a pull request for.
  ii) Click "Create Pull Request". GitHub Desktop will open your default browser to take you to GitHub.
 iii) On GitHub, confirm that the branch in the base: drop-down menu is the branch where you want to merge your changes.
      Confirm that the branch in the compare: drop-down menu is the topic branch where you made your changes.
  iv) Type a title and description for your pull request.
   V) To create a pull request that is ready for review, click "Click Pull Request".
